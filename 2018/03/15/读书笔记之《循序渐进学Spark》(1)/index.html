什么是Spark

Spark是一个基于内存计算的统一大数据处理框架。

Spark相对于Hadoop的优势

  • 中间结果：MapReduce通常将中间结果输出到磁盘上，以达到存储和容错的目的。而Spark通过 DAG 将多个 Stage 任务串联或者并行执行，并将中间结果缓存在内存中，而无需将中间结果存储在HDFS上。（内存比磁盘效率高的多）
  • 执行策略：MapReduce在数据Shuffle之前会花费大量时间进行排序，而Spark不需要对所有情景都进行排序。
  • 任务调度开销：MapReduce是为了处理长达数小时的批量作业而设计的，在特殊情况下，提交任务的延迟非常高。而Spark采用了事件驱动类库 AKKA 来启动任务，并通过 线程池复用线程 来避免线程启动以及切换带来的开销。
  • 容错性：RDD之间通过血缘关系来进行维护，一旦某个RDD失败了，就能通过父RDD自动重建，保证了容错性。
  • API：Spark的API更加丰富，支持多语言编程。

Spark架构

Application：用户编写的Spark应用程序，由一个或多个 Job组成 。提交到Spark后，Spark为Application分配资源，将程序转换并执行。

Client：提交应用的客户端。

Driver：执行Application中的main函数并创建SparkContext。是 应用逻辑的执行起点 ，负责 Task任务的分发和调度 。

ClusterManager：在 YARN 模式中为 资源管理器。在 Standalone 模式中为 Master（主节点），控制整个集群。

Worker：Standalone 模式中为 从节点 ，负责控制计算节点。YARN 模式中为 NodeManager。是集群中任何可以运行Application代码的节点，运行着一个或者多个 Executor进程。

Executor：在计算节点上执行任务的组件。是Application运行在Worker Node上的一个 进程，该进程负责运行 Task，并且负责将数据存储在内存或者磁盘上。

SparkContext：应用的上下文，控制应用的生命周期。

RDD：弹性分布式数据集，Spark的基本 计算单元。

DAG Scheduler：根据应用构建基于 Stage 的 DAG，并将 Stage 提交给 Task Scheduler。

Task Scheduler：将 Task 分发给 Executor 执行，并返回结果。

Job：包含很多 task 的并行计算，可以认为是 Spark RDD 里面的 action，每个 action 的计算会生成一个 job。用户提交的Job会提交给 DAGScheduler，Job会被分解成 Stage 和 Task。 Job是由Action算子触发生成的由一个或者多个Stage组成的计算作业。

Stage：一个Job被拆分成若干个Stage，每个Stage执行一些计算，产生一些中间结果。它们的目的是最终 生成这个Job的计算结果。而每个Stage是一个 task set，包含 若干个task，是一个task set对应的 调度阶段。每个Job会 根据RDD的依赖关系切分成很多Stage。

Task：Task是Spark中 最小的工作单元，在一个executor上完成一个特定的事情。

TaskSet：TaskSet是一组 关联 但相互之间没有 Shuffle依赖关系 的 集合。

Spark Env：线程 级别的上下文，存储运行时重要组件的应用
①SparkConf：存储配置信息。
②BroadcastManager：负责广播变量的控制及元信息的存储。
③BlockManager：负责Block的管理、创建和查找。
④MetricsSystem：监控运行时的性能指标。
⑤MapOutputTracker：负责Shuffle元信息的存储。

Spark运行流程

Client提交应用
↓
Master找到Worker，并启动Driver
↓
Driver向资源管理器（Yarn模式）或者Master（Standalone）申请资源，并将应用转化为RDD Graph
↓
DAG Scheduler 将RDD Graph转化为Stage的DAG提交给Task Scheduler
↓
Task Scheduler 提交任务给Executor执行。
