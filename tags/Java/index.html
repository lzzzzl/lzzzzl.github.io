JVM简单总结

什么是JVM

JVM即Java Virtual Machine，其是JRE的一部分，是虚构出的计算机，通过在计算机上模拟各种计算功能。JVM有完善的硬件架构，比如处理器、堆栈、以及寄存器等。Java的跨平台特性，就是由JVM支持实现。

JVM内存区域划分

JVM内部体系架构可以大致分为：类加载器子系统、运行时数据区、执行引擎。

类加载器子系统

每个JVM都有一个类加载器子系统，负责加载程序中的类和接口等类型，并赋予其唯一命名。JVM的两种类加载器包括：启动加载器和用户自定义类加载器。启动加载器是JVM实现的一部分，用户自定义类加载器是Java程序的一部分，其必须是ClassLoader的子类。

执行引擎

执行引擎即字节码执行引擎，对输入的字节码文件进行处理并解析，然后输出执行结果。主要三有种方式：芯片直接执行解释（第一代JVM），即时编译JIT（第二代JVM），自适应优化（结合第一代JVM和第二代JVM）

运行时数据区

  • 程序计数器（Program Counter Register）： 即当前线程所执行的字节码的行号指示器，每一个线程都有一个程序计数器。
  • 虚拟机栈： 这块内存区即为常说的”栈”，主要用于存放变量。
  • 本地方法栈： 本地方法栈与虚拟机栈类似，不同的是，虚拟机栈服务用的是Java方法，而本地方法栈服务用的是Native方法。
  • Java堆： 堆主要用于存放对象实例，数组内存等。堆被线程所共享，是垃圾回收GC的主要区域。可以被分为：新生代、老年代。新生代同时可以被分为Eden空间，From Survivor空间、To Survivor空间。
  • 方法区： 方法区又被称为”永久代”，其主要作用为存储已被虚拟机加载的类信息、常量、静态变量等数据。

读写锁ReadWriteLock

使用原因：

当使用重入锁或者内部锁的时候，读与写之间的操作都是串行的。由于读操作不对数据完整性产生破坏，因此读写锁有了用武之地。

ReadWriteLock特点

  • 读与读之间不互斥
  • 读与写之间读会阻塞写，写也会阻塞读
  • 写与写之间互相阻塞
