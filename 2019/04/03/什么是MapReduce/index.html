什么是MapReduce

MapReduce是适合处理海量数据的编程模型。 Hadoop能够运行以各种语言编写的MapReduce程序：Java，Ruby，Python和C ++。 MapReduce程序本质上是并行的，因此对于使用集群中的多台计算机执行大规模数据分析非常有用。

MapReduce如何工作？完整过程如下

整个过程经历了4个执行阶段，拆分(splitting), 映射(mapping), 派发(shuffling), 缩减(reducing)

让我们用一个例子加深理解

假如在你的Map Reduce程序中有如下一个输入数据


最终MapReduce任务的输出结果是:

key value
bad 1
Class 1
good 1
Hadoop 3
is 2
to 1
Welcome 1

这些数据经历了以下的阶段：

Input Splits

数据输入到MapReduce job后，会被分割为固定大小的块，被称为Input splits。Input splits是输入的一个块，会被一个map给消费掉。

Mapping

这是map-reduce程序执行过程中最为开始的一步。在这个阶段，每一个被拆分的数据都会被传入到一个mapping函数中，并产生输出值。在我们的例子中，一个mapping阶段的job会统计input splits(更多关于input-splits的细节会在后面给出)中每个词出现的频率，并且，会生成<词，频次>这一格式的l列表。

Shuffling

这一阶段会消费Mapping阶段产生的数据。它的任务是把Mapping阶段输出值中相关记录整合到一起。相同的词会被整合至一起，它们的频次也会汇总计算。

Reducing

在这一阶段，Shuffling阶段输出的数据会被聚合。它把Shuffling阶段的数据合并到一起然后返回一个单一的输出值。总的来说，这一阶段会汇总完整的数据集。

MapReduce如何工作

Hadoop把job分割成task，这里有两种task。

  1. Map tasks(Splits & Mapping)
  2. Reduce tasks(Shuffling, Reducing)

这是之前提到过的。

完整的运行过程(Map任务和Reduce任务都有执行)由如下两个组件所控制：

  1. Jobtracker：行为像master(负责所有已提交的job的完整执行)
  2. 多个Task Trackers：行为像slave，运行job

对于每个在系统中提交执行的job来说，这里有一个位于NameNode的Jobtracker，位于DataNode的多个tasktracker。

  • 一个job被分割成多个task，然后会运行在集群中的多个数据节点之中。
  • jobtracker的职责是通过调度运行在不同数据节点上的task，来协调活动。
  • 单个任务的执行由task tracker跟踪负责，task tracker位于每一个执行部分job的数据节点上。
  • Task tracker`s 的任务是发送变化报告给job tracker。
  • 此外，task tracker会周期性地发送心跳给到jobtracker，以此来通知它系统当前地状态。
  • 因此，job tracker持续跟踪每个job的总体进度。如何任务失败，job tracker可以在其他的task tracker上重新安排运行。
